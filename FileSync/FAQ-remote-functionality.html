How to use remote-functionality with FileSync in eclipse?

  1. Install RSE (http://www.eclipse.org/dsdp/tm/).
  2. Add a new RSE-connection with the RSE-eclipse-view "remote systems".
  3. Go to the FileSync-properties of your eclipse-project.
  4. Enter a valid remote-location-URI in the field "default target folder:"
    • Examples of valid URIs:
      • rse://RSE-connection-linux/home/username/remotefolder/for/filesync
      • rse://RSE-connection-windows/c:/remotefolder/for/filesync
    • "rse" is a fix prefix that tells FileSync to use the remote functionality.
    • The "host" (RSE-connection-linux/RSE-connection-windows) is the connection entered at step 2.
  5. If you have many files in a project and the connection to the remote system is slow, it can help to enable the "delayed copy/delete"-property.

How to change the unzip-command?

  1. Open the rse-eclipse-view "remote systems".
  2. Unfold the specific rse-connection.
  3. Right-click on the "*Shells"-label (e.g. "Ssh shells" or "Local shells")
  4. Choose Properties from the context-menu.
  5. Add the environment variable "FILESYNC_UNZIP_CMD" with the new unzip-command as value.
    • Examples: "unzip -o", "jar -xfv"
    • Important: The command must not prompt for user-input as FileSync can't provide this.

Delayed copy/delete fails. Why?

A delayed copy requires a program to decompress a zip-file. The default-command for linux on the remote-site is "unzip -o" and "jar -xfv" for windows. See How to change the unzip-command? if your server has a different program.

When the delayed copy/delete fails, take a look into the eclipse-view "error-log". There should be a description of the error or a link to a log-file.
